A-1
將 45 個 Domain 匯入個人 Obsidian

不是發佈於網站供所有人讀到相同內容,而是各自 pull 後由 AI 個人化消化、跨頁連結、持續更新——形成一份「知識的自愈系統」。集體 → 個人 → 變體 → 新知識 → 回饋集體。

KARPATHY LLM WIKI · 三層架構
Raw 不可變來源 · repo + domain/*.md（唯讀）
Wiki AI 維護的知識頁 · vault/wiki/
Schema 紀律設定檔 · vault/CLAUDE.md
Ingest·Query·Lint
A-2
三層架構 + 三個操作

Raw（唯讀來源）/ Wiki（AI 寫的個人筆記）/ Schema（規則檔）。三個操作:Ingest 消化新來源、Query 提問、Lint 健檢。「筆記是個人的壓縮,讀別人的筆記必然有損」——所以要建你自己的。

建置步驟:複製「想法檔」貼給 AI Agent

不需手動建立資料夾或執行 git——這些均由 AI 完成。使用者僅需複製完整的「想法檔(idea file)」貼進 Claude Code / Gemini CLI / Antigravity,由 AI 協作建立 vault。一致性靠複製不靠模型自律:schema 從範本逐字複製、idea file 尾部附 Karpathy 原文作為錨點。

完整建置說明(想法檔全文)在 BIM_MCP 知識站

1

裝 Obsidian

從 obsidian.md 下載安裝(免費)。先不用設定任何東西。

2

在 REVIT_MCP 資料夾啟動 AI Agent

就是你平常跑 MCP 的那個 clone——不用開新資料夾、不用第二份 clone。在那裡開 Claude Code(或 Gemini CLI / Antigravity)。還沒 clone 的人先走部署指南。

3

貼上「想法檔」建置說明核心:你唯一要做的

把完整想法檔整段貼進對話。AI 會自動:確認你在 repo 根目錄 → 建 vault/ → 在 vault/ 裡 git init → 從 templates/personal-vault/VAULT-CLAUDE.md 逐字複製 schema(Fixed Core 一字不改)→ 問你幾個個人化問題填 Personal 區(你的專業領域?最常用哪些 Domain?筆記語言粒度?)→ 跑驗收清單逐項回報。

4

第一次 Ingest

向 AI 說明你的專業(例如「我主要做消防審查」),由其從最相關的 Domain 開始消化——例如 fire-rating-check、corridor-analysis-protocol、smoke-exhaust-review。不一次處理全部 45 個。

建好之後:日常兩步

vault 建好後,日常只剩兩個動作——打開 Obsidian 看知識族譜、之後每次 pull + ingest 保持同步。

5

用 Obsidian 打開整個 REVIT_MCP 資料夾

開整個 repo(不是只開 vault/),Settings → Files and links → Excluded files 排除 MCP-Server/ 與 MCP/。graph view 同時顯示公共 domain 與你的 wiki——你的 [[連結]] 連到 domain 檔就是圖上的邊,公共＋個人合一的知識族譜。

↻

之後每次:pull + ingest

上游有更新(月小聚後通常會有)時,向 AI 輸入「pull 一下,ingest 有變動的」即完成更新——同一次 pull 亦同步更新 MCP 工具與 schema 範本。發現值得回饋者,可請 AI 協助整理提案。

操作你說AI 做
Ingestpull 一下,ingest 有變動的git pull → diff → wiki 更新 → 記 log
Query[任何問題]（可連 Revit 一起問）讀 index.md → 深入 → 存結果回 wiki
Lint健檢一下檢版本 / 矛盾 / 孤兒頁 / 可回饋項
A-4
日常只有三個動作

Ingest / Query / Lint。你做過的每次審查都 file 回 wiki 變案例——從「一次性查詢」變「知識累積」。Lint 發現「排煙檢討和外牆開口檢討應該互相關聯」,就變成下一個 PR。

為什麼是「升級」不是「加工具」

昨天:domain/ 都在上游網站上,大家看一樣的內容。今天:每個人 pull 回來,AI 為你個人化消化、跨頁連結、自動更新。

三點理由(出自 0612 vault 設計對話的 AI 評估):

  1. 數據底層已存在——45 個 domain 的 frontmatter 已有 related / referenced_by / tags,你要的是視覺化與個人化,不是從零建。
  2. 一致性有機制——複製 + schema_version + lint 自我修復,解決「多模型、多 CLI 建出來不一樣」。
  3. 閉環有路徑——/hj-pr-proposal 現成、每月小聚場地現成,回饋是制度化的日常。

直引:「我們不再只是在用工具,我們是在建立知識的自愈系統。」

三層架構各放什麼

層是什麼位置
Raw不可變來源repo 本身 + domain/*.md(唯讀)
WikiAI 維護的知識頁vault/wiki/(AI 寫)
Schema紀律設定檔vault/CLAUDE.md(從上游範本複製)

三個操作

  • Ingest 消化新來源
  • Query 提問
  • Lint 健檢

核心觀念:「筆記是個人的壓縮,讀別人的筆記必然有損」——所以要建你自己的;「wiki 內容刻意不一致是 feature,不是 bug」。

三個斜線指令

指令做什麼
/ingestpull 上游 → 讀 diff → 消化變動 → 更新 wiki/index → 記 log
/wiki查詢個人 vault wiki,可連 Revit 一起問,結果存回 wiki
/lint健檢:版本、矛盾、孤兒頁、可回饋項、schema 升級

進階:lint 找到「該連而未連的 Domain / 缺的法源 / 新應用」→ 用 /hj-pr-proposal 整理成 PR 提案,回饋上游。

紀律:個人操作永不寫入 vault/ 以外的檔案、永不 git clean -x、個人筆記永不 push、溯源記 source + source_version。

升級的四層敘事

  1. 從靜態網站到動態筆記——大家看一樣 → AI 為你個人化消化、跨頁連結、自動更新。
  2. 從一次性查詢到知識累積——傳統 RAG 每次從頭拼湊 → wiki 是持久資產,交叉引用已就位,每次審查 file 回 wiki 變案例。
  3. 從被動讀者到主動貢獻者——pull 完讀完就結束 → 「集體→個人→變體→新知識→回到集體」正向循環。
  4. 從分散知識到知識的自愈系統——零散筆記 → 自我健檢、自我修復、自我回饋。

第一次部署？跟著做就好

不需要懂程式、不需要懂 Git。
從零開始，把 Revit MCP 裝到你的電腦上。

1

用 Git Clone 把專案複製到你的電腦

Clone 是 Git 的「複製」功能。用 clone 而不是下載 ZIP，
以後專案更新時才能用 git pull 一鍵同步最新版。

基本版（推薦新手）：
1. 點開 github.com/shuotao/REVIT_MCP_study
2. 點綠色的 <> Code 按鈕
3. 複製 HTTPS 網址
4. 開啟終端機（按 Win + X → 選「終端」或「PowerShell」）
5. 逐行貼上以下指令（每行貼完按 Enter）：
cd Desktop git clone https://github.com/shuotao/REVIT_MCP_study.git

沒有 Git？ 前往 git-scm.com 下載安裝，一路按 Next 即可。

進階版（想貢獻程式碼）： 先在 GitHub 頁面右上角按 Fork，再 clone 你自己的 fork。
這樣未來可以提交 Pull Request 回饋給專案。

2

雙擊 setup.bat 開始自動安裝

打開桌面上的 REVIT_MCP_study 資料夾，找到 scripts 資料夾，裡面有一個 setup.bat。
雙擊它。 就這樣。不需要管理員權限。

它會自動幫你：
• 檢查電腦有沒有 Node.js → 沒有就自動安裝
• 檢查電腦有沒有 .NET SDK → 沒有就自動安裝
• 編譯 MCP Server（AI 跟 Revit 之間的翻譯官）
• 讓你選 Revit 版本 → 自動編譯 + 部署
• 自動設定 AI 客戶端（Claude Desktop / Gemini CLI / VS Code）

畫面會出現一個黑色視窗，跑完大約 3-5 分鐘。
選 Revit 版本時：上下方向鍵 移動、空白鍵 勾選/取消、Enter 確認。

或者讓 AI 幫你跑
AI

用任何 AI 工具執行一行指令

如果你已經有 VS Code / Gemini CLI / Claude Code 等 AI 工具，
直接貼上這行指令，AI 會幫你完成所有步驟：

powershell -ExecutionPolicy Bypass -File scripts/setup.ps1 -NonInteractive -RevitVersions "2024"

點擊上方指令即可複製。把 2024 改成你的 Revit 版本（2022/2023/2024/2025/2026）。
多版本用逗號："2024,2025"

3

打開 Revit，啟動 MCP 服務

安裝完成後，關閉 Revit（如果有開的話），重新打開。

1. 開啟 Revit → 載入任意專案
2. 上方功能列找到 MCP Tools 面板
3. 點擊 MCP 服務 (開/關) 按鈕
4. 看到「WebSocket 伺服器已啟動」就成功了
✓

開始用自然語言控制 Revit

打開你的 AI 工具，直接用中文跟它說你要做什麼。

你：幫我查 3 樓所有房間的面積
AI：已查詢完成，3F 共 12 間房間，總面積 486.3 m²...

你：把面積小於 20 平方的房間標紅色
AI：已將 3 間房間標記為紅色...

支援的 AI 工具：Claude Desktop / Gemini CLI / VS Code Copilot / Google Antigravity
setup.bat 已經幫你設定好了，直接開啟就能用。

遇到問題？

重新跑一次 setup.bat — 已完成的步驟會自動跳過
或到 GitHub Issues 回報
