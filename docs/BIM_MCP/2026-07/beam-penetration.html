GOTCHAS

三個技術陷阱

fork PR 舊檔貼回陷阱

GitHub 顯示 mergeable，textual merge 乾淨，但 PR 的 CommandExecutor.cs 是 fork 舊檔整個貼回：刪 12 個現役 case（門窗圖例、樑翻模、走廊分析等）。因方法在 partial class，build 照樣過、runtime 才壞。教訓：合併 fork PR 前先 diff dispatcher case 清單。

PR 切片的依賴閉包

從大分支切出來的 PR 可能呼叫沒隨附的方法；本次 PR 呼叫了 7 個未隨附的 helper 方法，無法單獨編譯。教訓：合併前要追依賴閉包，缺的從同作者原分支補。

WebSocket 大封包截斷

單次 ReceiveAsync + GetString(count) 遇分片 frame 會截斷 JSON；改 MemoryStream 累積至 EndOfMessage 才轉字串。此修正已 cherry-pick 進 main。
