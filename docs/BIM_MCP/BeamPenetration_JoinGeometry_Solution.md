# Revit 穿梁套管標註：幾何接合與實體邊緣判定解決方案

## 1. 問題背景
在開發「穿梁套管檢核與自動標註」的 MCP 應用時，標註線必須精準鎖定在「套管外緣」與「梁的物理端點（被柱子或正交梁截斷的表面）」。
但在 Revit 中，結構元素（如混凝土梁與梁、梁與牆）會自動發生**幾何接合 (Join Geometry)**。這會帶來兩個嚴重的標註偏移災難：

1. **平行牆體干擾**：若主梁與平行於它的牆體接合，兩者的實體幾何 (Solid) 會融合。若無差別讀取實體頂點，平行牆較長的邊界會被誤判為主梁的端點，導致尺寸線被拉伸到牆的盡頭。
2. **正交梁接合偏移**：當主梁端點與正交梁接合時，主梁的「截斷面」會因融合而消失。若退回使用「正交梁中心點向兩側推算半寬 (W/2)」的數學算法，只要正交梁在建模時有做偏心對齊或扣減，中心點減半寬算出的邊緣就會與實際表面產生數公分的視覺誤差。

---

## 2. 解決方案：降維排隊與實體投影演算法

為徹底解決上述問題，我們放棄了單一的推算邏輯，改採「分而治之」的幾何降維演算法，實作於 `CreateSequentialDimensions` 方法中。

### 步驟一：處理大梁本身的端點 (Boundary Nodes)
專注處理梁遇到柱子或牆壁而產生的物理截斷面。
1. **遍歷實體表面**：讀取大梁所有實體的平面 (`PlanarFace`)。
2. **法向量過濾 (Dot Product)**：計算面的法向量與梁中心線的內積 (Dot Product)。
   - 僅保留 `dot > 0.8`（法向量近乎平行於梁方向）的面，這代表它是「正對著」梁的截斷面。
   - 完美排除平行牆壁（法向量垂直於梁，dot 近乎 0）的干擾。
3. **極值距離判定**：將過濾出的面分為靠近起點 (p0) 與靠近終點 (p1) 兩組，並各自取**「距離端點最近」**的面作為真實物理邊界。

### 步驟二：處理套管節點 (Sleeve Nodes)
將套管中心點投影至大梁中心線，並向左右推算其半徑 (D/2)，建立具有左邊緣 (LeftEdge) 與右邊緣 (RightEdge) 的套管節點。

### 步驟三：處理正交梁節點 (Orthogonal Beam Nodes) - 實體頂點投影法
解決中心點推算誤差與接合面消失的問題，直接對「正交梁」的實體下手。
1. **獲取正交梁實體**：不管主梁面消不消失，直接讀取干涉的正交梁實體 (Solid)，即使它位於連結模型 (Linked Model) 中也同樣適用（解決唯讀問題）。
2. **全實體頂點投影**：將正交梁幾何的所有頂點 (`edge.Tessellate()`)，垂直投影到主梁的中心線上。
3. **建立足跡 (Footprint)**：找出這些投影點中的最小值 (`minParam`) 與最大值 (`maxParam`)。
   - `minParam` 即為正交梁物理幾何上最靠近起點的「左邊緣」。
   - `maxParam` 即為最靠近終點的「右邊緣」。
   - 完全捨棄 W/2 算法，取得 100% 準確的物理視覺邊界。

### 步驟四：一維降維打擊的「排隊畫線法」
將複雜的 3D 空間面相問題，轉化為 1D 線段的排隊問題。
1. 將所有節點（端點、套管、正交梁）**依據其中心點距離 (CenterDist) 由小到大排序**。
2. 兩兩配對，由「左側節點的右邊緣 (RightEdge)」畫線至「右側節點的左邊緣 (LeftEdge)」。
3. 只要套管在正交梁的左側，必然會畫到正交梁的 `LeftEdge`（靠近套管的面）；反之則會畫到 `RightEdge`。
4. 完美避免標註跨越實體或標註到錯誤側面的問題。

---

## 3. 結論
透過結合**「法向量過濾」**與**「實體極值投影法」**，我們成功避開了 Revit 幾何接合機制帶來的黑箱盲區。這套演算法不僅能在主模型內發揮作用，對於無法修改接合狀態的**連結模型 (Linked Model)** 同樣具備極高的強健性 (Robustness)。
