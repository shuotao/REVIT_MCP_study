## 為什麼你不能直接 git pull

你的 fork 裡有 Revit 2020 專用的修改（`Floor.Create` 條件編譯、`IdType` 相容性、`CreateDetailLines` partial class 調整）。如果直接 `git pull upstream main`，這些修改會被 2022-2026 的版本覆蓋，造成：

- `Floor.Create` 簽名不相容 → 編譯失敗
- `CreateDetailLines` 在 partial class 之間衝突 → 合併衝突
- `RevitMCP.csproj` 沒有 `Release.R20` → 無法 build

**但是**，專案裡有大量跟 Revit 版本無關的內容（知識、規範、工具定義），這些你可以也應該同步。

## 操作步驟

### 第一次設定（只做一次）

```bash
# 1. 進入你的 fork 目錄
cd REVIT_MCP_study

# 2. 加入 upstream remote（如果還沒加過）
git remote add upstream https://github.com/shuotao/REVIT_MCP_study.git

# 3. 確認 remote 設定正確
git remote -v
# 應該看到：
# origin    https://github.com/Jacky820507/REVIT_MCP_study.git
# upstream  https://github.com/shuotao/REVIT_MCP_study.git
```

### 每次同步（用腳本）

```bash
# 執行選擇性同步腳本
bash scripts/sync-knowledge-only.sh
```

腳本會：
1. 自動 `git fetch upstream`
2. 列出要同步的安全目錄
3. 問你確認（按 `y`）
4. 只拉知識層的檔案
5. 顯示變更清單

同步完後：

```bash
# 檢查變更
git diff --cached --stat

# 確認沒問題後 commit
git commit -m "chore: sync knowledge layer from upstream"

# 推到你的 fork
git push origin main
```

### 每次同步（手動方式）

如果腳本有問題，你也可以手動一個一個拉：

```bash
# 取得最新資料
git fetch upstream

# 只拉特定目錄
git checkout upstream/main -- domain/
git checkout upstream/main -- .claude/skills/
git checkout upstream/main -- .claude/hooks/
git checkout upstream/main -- .claude/settings.json
git checkout upstream/main -- CLAUDE.md
git checkout upstream/main -- GEMINI.md
git checkout upstream/main -- docs/
git checkout upstream/main -- MCP-Server/src/tools/

# 檢查 + commit
git diff --cached --stat
git commit -m "chore: sync knowledge layer from upstream"
git push origin main
```

## 同步後要做的事

### 1. 檢查 CLAUDE.md 的數字

CLAUDE.md 裡有一些硬編碼的數字，同步後可能跟你的實際狀況不同：

```bash
# 你實際有幾個工具？（可能跟主線的 58 不同）
grep -c "case \"" MCP/Core/CommandExecutor.cs

# 你實際有幾個 Skill？
ls .claude/skills/*/SKILL.md | wc -l

# 你實際有幾個 Domain？
ls domain/*.md | wc -l
```

如果數字不一致，在你自己的 CLAUDE.md 裡修正（不需要提 PR）。

### 2. 確認新的 Skill 是否適用於 Revit 2020

新同步的 Skill 可能引用了 2022+ 才有的工具。檢查方式：

```bash
# 看有沒有新的 Skill
git log --oneline --diff-filter=A -- .claude/skills/

# 讀新 Skill 的工具清單，確認你的 CommandExecutor 裡有對應的 case
```

如果某個 Skill 引用的工具你沒有實作，**不需要刪除 Skill**，
只是那個 Skill 在你的環境裡無法完整執行。

### 3. 確認 Hook 是否正常

```bash
# 測試 hook 偵測腳本
echo '{"tool_name":"Bash","tool_input":{"command":"git merge test"}}' | \
  .claude/hooks/detect-claudemd-trigger.sh

# 應該看到 JSON 輸出，包含「合併外部 PR」
```

## 你想提 PR 時的注意事項

你的 PR 應該只包含：

### ✅ 歡迎提交
- `domain/*.md` — 新的法規 SOP
- `.claude/skills/*/SKILL.md` — 新的 Skill 定義
- `domain/lessons.md` — 新的開發經驗

### ⚠️ 需要討論
- `MCP/Core/CommandExecutor.cs` — 你的版本有 2020 特定的條件編譯，
  如果你要提交新的 command，建議**只提交新增的方法**，不要提交整個檔案
- `MCP-Server/src/tools/*.ts` — 工具定義可以提，但要確認 C# 端有對應實作

### ❌ 不要提交
- `RevitMCP.csproj` 的版本修改
- `bin/`、`obj/`、`scratch/` 等暫存目錄
- 你的個人設定檔

### PR 提交前的檢查

```bash
# 確認你只 add 了該提交的檔案
git diff --cached --name-only

# 確認沒有夾帶二進位檔案
git diff --cached --name-only | grep -E '\.(dll|exe|pdb|bin|obj)'

# 確認沒有夾帶設定檔
git diff --cached --name-only | grep -E '(settings\.local|\.env|config\.json)'
```
