# Toposolid 樓板投影整地試跑設計

## 目標

在 Revit 2024 中，以目前選取的兩片樓板作為整地控制面，將樓板平面投影範圍內的設計地形完成面貼齊各樓板底面，並讓 Revit 能以既有地形與設計地形的差異回報 CUT、FILL 與淨土方。

本次僅驗證 `Footprint Only`。Offset 外延、指定坡度、指定邊界與即時自動更新不納入本次實作。

## 已確認元素

- 既有地形：Toposolid `6278563`
- 控制樓板：Floor `7512796`
- 控制樓板：Floor `7512816`
- 執行環境：Revit 2024

執行時仍須重新驗證選取集合與元素類別，避免使用者在規格確認期間變更選取或刪除元素。

## 使用者操作介面

新增 MCP 工具 `grade_toposolid_to_floors`，參數如下：

- `toposolidId`：既有或目前設計地形的元素 ID。
- `floorIds`：一個以上控制樓板 ID。
- `mode`：本次固定支援 `footprint_only`。
- `targetFace`：本次固定為 `bottom`。
- `allowPhaseSetup`：預設 `true`，自動設定整地所需階段；傳 `false` 時不修改階段，僅回報所需變更。
- `updateExisting`：預設 `false`；本次建立新的整地結果，不覆寫先前結果。

工具成功時回傳：原地形 ID、設計地形 ID、控制樓板 ID、CUT、FILL、淨土方、修改點數、警告、可供後續更新使用的關聯識別碼，以及 `Timing` 分段計時（總毫秒數與各階段耗時）。

## 幾何規則

1. 取得每片樓板的有效底面與水平投影邊界。
2. 樓板投影範圍內，設計地形頂面目標高程等於該 XY 位置的樓板底面高程。
3. 樓板範圍外保持原地形不變。
4. 在樓板邊界加入控制點，避免修改擴散到投影範圍外。
5. 若樓板底面有坡度，依底面平面或面幾何計算各控制點高程，不以單一平均高程取代。
6. 若兩片樓板投影重疊且底面高程不同，工具中止且不提交交易，要求使用者處理衝突。
7. 若樓板投影超出地形邊界，僅處理與地形相交的區域，並回傳警告。

## Cut/Fill 與階段

Revit 的 CUT/FILL 以既有地形與新階段設計地形比較。工具執行前先檢查：

- 原 Toposolid 必須可作為既有地形。
- 若其 `Phase Created` 不是較早階段，工具預設自動調整（`allowPhaseSetup` 預設 `true`）：原地形改為較早階段建立、目前階段拆除，設計副本建立於目前階段，與 Revit 原生整地行為一致。傳 `false` 時工具中止並回報所需變更。（2026-07-03 依使用者決策由「預設詢問」改為「預設自動」：一般使用者的地形本來就建立在新建階段，工具應代為判斷與設定。）
- CUT、FILL 數值以 Revit 重新生成後的內建參數為準，不以自製網格體積取代正式報表值。

## 交易與回復

- 使用 `TransactionGroup` 包住前置檢查、整地建立與結果讀取。
- 任一幾何錯誤、階段錯誤或 Revit 失敗訊息都回滾整組交易。
- 不刪除控制樓板。
- 不永久改寫原地形幾何；以 Revit 的整地／階段關係保留既有狀態。
- 回傳新建或修改元素 ID，讓使用者可立即檢查與刪除試跑結果。

## 執行效能與記錄機制（2026-07-03 新增）

效能目標：整地只是整個建模流程上百步中的一步，單次執行應在數秒內完成；超標時以記錄數據定位瓶頸，逐次修正。

- 工具回應包含 `Timing`：`TotalMilliseconds` 與 `Stages`（各階段名稱與毫秒），階段涵蓋元素驗證、樓板底面擷取、設計副本與階段設定、邊界射線取樣、頂點分類、兩輪校準寫入、驗收檢查、CUT/FILL 讀取與各交易提交。
- 每次執行（成功與失敗皆記）附加一筆 JSON Lines 至 `%APPDATA%\RevitMCP\grading-performance.jsonl`，欄位含時間戳、模型名稱、成敗、錯誤訊息、元素 ID、修改點數、CUT/FILL 與分段計時。
- 記錄寫入失敗時靜默略過，不得影響整地流程。
- 優化候選（依記錄數據決定順序）：邊界射線取樣快取、頂點比對空間格網化、離散化弦長調整、Regenerate 次數合併。

## 更新策略

本次採隨叫隨更新，不註冊 `IUpdater`。後續樓板範圍或高程改變時，使用者重新執行工具；工具依關聯識別碼重建設計地形。即時自動更新留待幾何規則與試跑結果確認後再設計。

## 錯誤處理

以下情況不修改模型並回傳繁體中文錯誤：

- ID 不存在或類別不是 Toposolid／Floor。
- 樓板沒有可用的底面或投影邊界。
- 投影範圍完全不與地形相交。
- 樓板投影重疊且目標高程衝突。
- 原地形階段不符合 CUT/FILL 計算前提。
- Revit 無法建立有效 Toposolid 或整地關係。

## 測試與驗證

1. 單元測試先驗證參數、類別、重疊衝突與單位轉換，並先觀察測試因功能不存在而失敗。
2. 建置 `Release.R24`，確認 `RevitMCP.dll` 更新。
3. 因 Revit 執行中無法覆寫已載入 DLL，部署時須先關閉 Revit，再重新啟動並恢復測試模型。
4. 重新選取一個 Toposolid 與兩片樓板後執行工具。
5. 驗證設計地形在樓板投影內貼齊底面、投影外未改變，並檢查 CUT/FILL 參數與元素 ID。
6. 若 Revit 2024 API 無法以支援方式建立 Graded Region 關係，停止修改並回報限制，不以 UI 模擬或不穩定反射 API 取代。

## 模式 Roadmap（2026-07-02 討論定案）

整地行為分三種模式，對應工具的 `mode` 參數；本次試跑只實作第 1 種，其餘兩種為已討論確認的後續方向。

| # | 模式 | `mode` 值 | 狀態 | 說明 |
|---|---|---|---|---|
| 1 | 樓板投影直貼 | `footprint_only` | 已實作（2026-07-02） | 樓板底面水平投影範圍內，地形完成面直接貼齊樓板底面；範圍外不動。投影邊界會形成垂直落差，如同開挖後尚未放坡的擋土邊界。 |
| 2 | Offset 外延銜接 | `offset_transition`（暫名） | 規劃中 | 在樓板投影外再外延一段使用者指定的 Offset 範圍，於此範圍內從樓板底面高程漸變回原地形高程，消除邊界垂直斷面。 |
| 3 | 指定坡度銜接 | `slope_transition`（暫名） | 規劃中 | 使用者指定目標坡度（例如 1:12）與最大延伸距離，從樓板投影邊界以固定坡度放坡回原地形；實際延伸距離由高差與坡度求得，超過上限則回報警告。 |

模式 2、3 的參數草案（實作前需再確認單位與命名）：

- `offsetDistance`：模式 2 的外延距離。
- `slopeRatio`：模式 3 的目標坡度，以 `1:n` 表達（例如 `1:12`）。
- `maxExtension`：模式 3 的放坡延伸距離上限。
- `boundaryEdges`：可選——只對指定邊界套用銜接，其餘邊界維持直貼。

其他後續擴充（未排程）：

- Offset 加坡度的組合模式（先平接一段再放坡）。
- 全部邊界或指定邊界分別指定銜接方式。
- 樓板變更後的 `IUpdater` 即時更新。

## 紀錄、標註與輸出 Roadmap（2026-07-02 討論定案，2026-07-03 補充方案管理）

### 核心設計：方案單元

每個方案 = 一片設計地形 + 一個鎖定 3D 視圖 + 登記簿一筆。

- 設計地形：工具每次執行建立新的設計副本，原地形永遠不動，方案天生共存、互不覆蓋。
- 鎖定 3D 視圖：每方案複製一個 3D 視圖，隔離顯示該方案的設計地形與控制樓板；Revit 規定 3D 視圖鎖定後才能放 Spot Elevation，此視圖同時是標註載體、截圖來源與方案書籤（回看方案＝切換視圖）。
- 登記簿一筆：欄位見「登記欄位完整度清單」。

### Design Option 定位

多方案挖填紀錄不以 Design Option 作為資料載體，理由：

1. Revit 公開 API 無法建立或切換 Design Option，無法自動化。
2. CUT/FILL 依階段比較，疊加選項後明細表與視圖需依選項過濾，易誤讀。
3. 僅 Primary Option 會出現在一般視圖，次要方案截圖需先調整視圖設定。

定位為「帳本自己記，展示交給 Design Option（選用）」：資料以設計地形上的 Extensible Storage 為準；使用者需要並排比較時，再手動將設計地形放入 Design Option。

### 多方案紀錄與輸出

- 方案登記簿：擴充既有 `AssociationId` Extensible Storage，每次執行記錄一筆（欄位見下）。
- Excel 比較表：一列一方案，數值欄位加嵌入該方案視圖的截圖，讓數值與畫面同表比對；沿用 `export_smoke_review_excel` 的既有基礎設施。
- 結果截圖：以 `ImageExportOptions` 匯出方案 3D 視圖 PNG，嵌入 Excel 並同時輸出獨立圖片檔。

### 登記欄位完整度清單

每筆方案記錄應包含：

1. 方案識別：名稱、時間戳、模型檔名、AssociationId。
2. 參數：模式、Offset、坡度、控制樓板 ID、原地形與設計地形 ID。
3. 土方：CUT、FILL、淨土方；後續加鬆實方三本帳（自然方、鬆方、壓實方）。
4. 幾何指標：最大挖深與最大填高（擋土設施需求與水保簽證門檻）、擾動面積（山坡地水保計畫門檻指標）、各樓板面積與底面高程。
5. 基準與追溯：高程基準（海拔或專案基準點）、執行時的警告訊息。

### 方案還原策略

- 方案間僅樓板高程不同（最常見）：登記簿已記錄各樓板高程參數，還原＝工具寫回參數，一鍵完成。
- 方案間樓板輪廓不同：API 完整還原草圖成本過高；改以 Design Option 作手動幾何容器（使用者自行放入並切換），或接受「數值與截圖留檔、幾何不還原」。

### 樓板與圖面標註

- 樓板高程標註：以 `NewSpotElevation` 建立 Spot Elevation，隨模型變動更新；不以文字註記取代。3D 視圖須先鎖定才能放置（配合方案視圖設計）。
- 樓板面積：讀取內建面積參數（現有 `get_element_info` 已支援，無需新工具）。
- 周邊關鍵高程：於地形邊界與放坡起訖點建立 Spot Elevation。

### 樓板底面通用化（2026-07-03 討論定案）

目標：儘可能支援各式樓板情況，閘門只擋「算不對」的，不擋「長得不一樣」的。

- 曲線邊板：已支援（邊界離散化以 300 mm 最大弦長處理圓弧與樣條）。
- 斜板：2026-07-03 已放寬至 15°；更陡的板改為「取最朝下的面」選面邏輯後即可全面支援。
- 變厚度板、形狀編輯過的板（底面多面或翹曲）：v2 架構——放棄「單一平面底面」假設，改用「樓板輪廓投影＋對樓板實體垂直射線逐點取底面 Z」，可支援任意底面幾何；與現況地形取樣為同一套射線技術。
- 原則：接受的案子必須給出可驗證的正確數字（2 mm 驗收閘門不變）；尚無把握算對的幾何，明確拒收並說明原因，不默默輸出錯誤土方。

### 進階功能

- 平衡高程反求：指定淨土方目標（預設趨近 0），以二分法迭代樓板高程求挖填平衡；放坡定案前決策價值最高。
- 鬆實方係數：使用者設定鬆方與壓實係數，登記簿同時呈現自然方、鬆方、壓實方三本帳，避免帳面平衡實際不平衡。
- 挖填熱區圖：地形差異上色（紅為挖、藍為填、深淺表深度），快速定位土方集中區。
- 邊界剖面：沿放坡邊界以 `create_section_view` 自動建立剖面，顯示原地形線與設計地形線，供送審與溝通。
- 土方計算書格式：Excel 匯出排版成主管機關（水保、雜項執照）要求的斷面法或方格法計算書。

### 實作優先順序

方案登記簿與 Excel 匯出 → 平衡高程反求 → 方案視圖（鎖定 3D、截圖、嵌入 Excel）與挖填熱區圖 → 標高標註 → 方案還原 → 鬆實方係數與土方計算書。
