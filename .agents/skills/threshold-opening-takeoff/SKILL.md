# 門檻開口統計

使用此 Skill 依照使用者指定的排除規則，統計與房間相關的門或窗。目前主要用途是不鏽鋼門檻數量統計；窗戶也先保留相同的房間/開口統計模式，供未來工作流使用。

## 核心規則

門檻統計時，只要門的任一側房間符合排除規則，該門就要整樘排除。不要只檢查主要計入房間。

一定要同時檢查：

- `FromRoomName`
- `ToRoomName`

只要任一側符合排除清單、排除關鍵字，或使用者指定的自訂條件，就將整樘門自統計中移除。

## 工作流程

1. 確認目標類別：
   - 門：使用 `get_room_door_counts`。
   - 窗：使用 `get_room_window_counts`。
2. 要求回傳明細列：
   - 門：`includeDoorDetails: true`。
   - 窗：`includeWindowDetails: true`。
3. 使用 `query_elements_with_filter` 取得類型參數：
   - `Doors` 類別：至少要求 `類型備註`、`類型`、`族群與類型`。
   - `Windows` 類別：模型可用時，要求對應的類型備註/類型欄位。
4. 依 `ElementId` 建立唯一開口清單。
5. 對 `FromRoom` 與 `ToRoom` 兩側套用房間排除規則。
6. 對類型名稱與族群/類型名稱套用類型或族群排除規則。
7. 依使用者要求的鍵值彙總保留的開口：
   - 門檻統計預設使用門的類型參數 `類型備註`。
   - 若 `類型備註` 為空白，回報為 `空白`。
8. 只回報本回合工具回傳資料可驗證的統計數量。

## 預設門檻規則

從使用者提供的規則開始，再允許追加或覆寫：

- 門任一側房間包含明確指定的排除房名時，排除該門。
- 使用者要求時，門任一側房間包含 `機房`、`廁所`、`管道間` 等關鍵字時，排除該門。
- 使用者要求時，門的類型/族群文字包含 `管道間` 或 `鐵捲門` 時，排除該門。
- 依門類型參數 `類型備註` 彙總；在此工作流中，這個欄位視為門編號。

不要把這些預設值當成所有專案通用規則。若使用者在對話中變更規則，回報最終結果前要先簡短列出目前生效的規則。

## 資料誠實

規則變更後，不可沿用先前統計數字。必須重新查詢 Revit，或明確說明該數字已過期。

每個回報的數量、類型備註、房名與 `ElementId`，都必須能追溯到本回合工具回傳資料。如果 Revit MCP 工具無法使用，只能提供通用方法，不可編造專案數量。
