L1276: 本 PR 實作了「剖面自動編排與命名」功能，能夠自動根據平面圖上選取的剖面幾何座標，進行符合看圖習慣的空間排序，並批次重新編號命名。
L1278: 這解決了以往在 Revit 中，人工為大量剖面調整順序與更名時極度繁瑣且容易跳號、重名的痛點。
L1280: 2. 🛠️ 解決的核心技術問題 (Problems Solved &amp; Technical Highlights)
L1281: 剖面標記類型相容 (Revit API 類型容錯)：
L1282: 問題：使用者在平面圖上選取的是「剖面標記 (Section Marker)」而非真正 View 物件，直接修改名稱會拋出 Null 錯誤。
L1283: 解法：在 C# 端 RenameView 交易中加入容錯 ── 當選取物件非 View 時，自動利用其同名特性，透過名稱搜尋匹配模型中的 View 物件進行重命名。
L1284: 排帶式空間掃描排序演算法 (Row-major Grouping)：
L1285: 問題：若只按單一軸線排序會不符合看圖習慣；若使用距離相鄰聚類，密集剖面會像鏈條一樣把不同橫排融合成一大群 (Chaining Bug)。
L1286: 解法：實作「固定高度帶分群」，將 Y 座標依區間精準分出橫排（Row 1~4），在每一排內部再依 X 座標（由左至右）進行排序，實現完美的「由上至下、由左至右」掃描順序。
L1287: 兩階段雙防衝突命名 (Two-Pass Renaming)：
L1288: 問題：直接重命名會與同組現有視圖名稱衝突，導致 Revit 拋出 Name must be unique 錯誤而中斷。
L1289: 解法：第一階段將所有選取剖面改為唯一的臨時名稱（帶時間戳與索引），完全釋放目標命名空間；第二階段再改為正式遞增名稱。
