1. 🚀 功能概述 (What does this PR do?)
本 PR 實作了「剖面自動編排與命名」功能，能夠自動根據平面圖上選取的剖面幾何座標，進行符合看圖習慣的空間排序，並批次重新編號命名。
----------------------------------------
[Paragraph 202]
這解決了以往在 Revit 中，人工為大量剖面調整順序與更名時極度繁瑣且容易跳號、重名的痛點。
----------------------------------------
[Paragraph 203]
2. 🛠️ 解決的核心技術問題 (Problems Solved &amp; Technical Highlights)
剖面標記類型相容 (Revit API 類型容錯)：
問題：使用者在平面圖上選取的是「剖面標記 (Section Marker)」而非真正 View 物件，直接修改名稱會拋出 Null 錯誤。
解法：在 C# 端 RenameView 交易中加入容錯 ── 當選取物件非 View 時，自動利用其同名特性，透過名稱搜尋匹配模型中的 View 物件進行重命名。
排帶式空間掃描排序演算法 (Row-major Grouping)：
問題：若只按單一軸線排序會不符合看圖習慣；若使用距離相鄰聚類，密集剖面會像鏈條一樣把不同橫排融合成一大群 (Chaining Bug)。
解法：實作「固定高度帶分群」，將 Y 座標依區間精準分出橫排（Row 1~4），在每一排內部再依 X 座標（由左至右）進行排序，實現完美的「由上至下、由左至右」掃描順序。
兩階段雙防衝突命名 (Two-Pass Renaming)：
問題：直接重命名會與同組現有視圖名稱衝突，導致 Revit 拋出 Name must be unique 錯誤而中斷。
解法：第一階段將所有選取剖面改為唯一的臨時名稱（帶時間戳與索引），完全釋放目標命名空間；第二階段再改為正式遞增名稱。
3. 📂 修改與新增的檔案 (Modified Files)
MCP/Core/CommandExecutor.cs
移出視圖相關代碼，進行類別解耦。
MCP/Core/Commands/CommandExecutor.ViewOps.cs (新增)
以 partial class 實作視圖模組，新增 RenameView 工具，並包含剖面標記轉同名視圖的容錯檢索邏輯。
domain/section-auto-numbering.md (新增/更新)
撰寫完整的剖面編排 SOP 與開發文件，詳細記錄幾何分帶演算法與兩階段重命名策略。
4. 🧪 測試與驗證結果 (How was this tested?)
測試環境：Revit 2023、本機 Node.js WebSocket 客戶端。
測試案例：
框選 4F 平面圖上共 35 個 剖面標記。
排序驗證：第一排 (最上方) 依左至右被精準命名為 1~10，接著依序編至第四排 (最下方) 右側的 35，完全符合物理幾何位置。
效能與穩定度：Two-Pass 共 70 次重新命名交易在同一 Revit 會期中順利完成，無任何 Name Duplicate 報錯，Revit 端視圖及瀏覽器名稱皆即時同步更新。" data-view-component="true" class="dropdown-item btn-link">

本 PR 實作了**「剖面視圖基準線自適應調整」**功能 (`adjust_section_datums`)。
本功能主要為了解決 Revit 剖面視圖中，網格線 (Grids) 與樓層線 (Levels) 的 2D 範圍與氣泡長度預設凌亂、手動調整耗時的問題。外掛會自動計算視圖的裁剪框 (Crop Box) 極值，並將基準線調整至適當長度與顯示狀態，且支援多視圖的批次排版處理。
---
## 🛠️ 主要變更與實作細節 (Key Changes)
### 1. 核心 API 路由與命令擴充
* **新增命令** `adjust_section_datums`。
* **模組化程式碼**：將視圖基準線幾何排版邏輯獨立封裝於專屬的 `MCP/Core/Commands/CommandExecutor.ViewOps.cs` 中。
### 2. 視圖裁剪框 (Crop Box) 自動處理
* 實作 `EnsureAndGetCropBox` 機制：如果剖面視圖的裁剪框未啟用，外掛將會自動將其啟用，並以其作為基準線延伸計算的幾何邊界。
### 3. 網格線 (Grids / 柱線) 2D 範圍與單側氣泡調整
* 自動將 Grids 端點轉換為剖面視圖特定的 2D 範圍（View-Specific）。
* 將網格線在垂直方向上，延伸超出裁剪框（Crop Box）上下邊界 **150 mm**。
* 控制網格氣泡顯示：**僅在上方顯示氣泡，下方氣泡自動隱藏**。
### 4. 樓層線 (Levels) 字元寬度自適應與雙側氣泡調整
* 自動將 Levels 端點轉換為剖面視圖特定的 2D 範圍（View-Specific）。
* **動態長度適應演算法**：讀取每個 Level 樓層名稱（例如 `4FL-1200` 與 `100FL-30000`）的字元長度，並結合當前視圖的 `Scale`（比例尺分母），動態換算出模型物理偏移量，確保文字與氣泡突出邊界的距離適中且不重疊。
* 控制樓層氣泡顯示：**雙側皆顯示氣泡**。
### 5. 多版本相容性優化 (Revit 2022 - 2026)
* 由於 Revit 2023 無 `CropRTransform` 屬性，本功能改用 `BasisX`, `BasisY`, `BasisZ` 與 `Origin` 手動建構局部座標 Transform 矩陣，保證在 Revit 2022 至 2026 各版本中皆能完美執行。
