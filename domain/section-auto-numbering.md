# 剖面自動編排與命名標準作業程序 (Section Auto-Numbering SOP)

## 1. 目的
自動化地對 Revit 中選取的剖面視圖（或剖面標記），根據其在平面上的座標位置進行順序編排，並統一套用自訂前綴進行重新命名。

## 2. 適用對象
- Revit 中的剖面視圖 (ViewSection) 或剖面標記 (Section Marker)。
- 工程師希望將平面上散亂的剖面符號（如：剖面 1、剖面 5、剖面 2）依照特定方向（如由左至右）整理為有條理的名稱（如：剖面-01, 剖面-02, 剖面-03）。

## 3. 前置準備
- Revit 專案已開啟，且使用者**已在視圖中手動選取**需要重新命名的剖面標記。

## 4. 作業流程 (AI 執行步驟)
1. **取得選取元素與座標**：
   - 呼叫 `get_selected_elements` 取得目前選取的元素。
   - 過濾出 `Category` 為 "Views", "視圖", "Sections", "剖面" 的元素，並取得其回傳的 `Origin` (X, Y, Z) 坐標。
2. **前綴名稱確認**：
   - AI 需檢視選取元素的現有名稱，若所有名稱具有相同的共同前綴（如都是 `A-` 開頭，或都是 `剖面` 開頭），則推論為預設前綴。
   - **強制詢問使用者**：如果無法推論出前綴，或者為了保險起見，AI **必須在文字對話中反問使用者**：「請問您要為這些剖面使用什麼前綴？（例如：`剖面`，將會產生 `剖面1`, `剖面2`...）」。
   - 等待使用者提供前綴後再繼續執行。
3. **單一方向排序**：
   - 根據使用者指定的排序方向（若使用者未指定，預設採用 **X 座標由左至右**，即數值由小到大）。
   - 對所有選取的剖面元素依照該座標軸的數值進行排序。
4. **批次重新命名**：
   - 依照排序結果，從 1 開始遞增編號。
   - 針對每個剖面，呼叫 `rename_view` 工具，傳入 `viewId` 與組合後的新名稱 (`newName` = 前綴 + 數字編號，如：`剖面1`)。

## 5. 常見問題與處理
- **名稱重複衝突**：如果新名稱與專案中既有的其他剖面名稱衝突，Revit 可能會報錯（名稱已在使用中）。若發生，可暫時在名稱後方加上亂數後綴（如 `剖面1_TEMP`），等全部改完再進行第二次正名。
- **無法取得座標**：若是選取到了非剖面物件，則跳過處理。
