## 🔍 詳細分析

### 1. [Potato] MCP 一鍵導出 (Native API)
*   **適合使用角色**：**營造下包 / 機械加工廠**。
*   **技術核心**：使用 Revit 2024 原生 PDF 引擎。
*   **可控細節**：檔名規則、自動路徑、隱藏參考平面、取消藍色超連結框。
*   **送審適用度**：適合高頻率的「零件圖」輸出，因為它實現了真正的「零手動」導出。
*   **可擴充性**：**最強**。可根據不同零件類別，自動將 PDF 存入對應的發包資料夾（例如：A 廠、B 廠）。

### 2. [guRoo] 批次導出 (Sheets PDF)
*   **適合使用角色**：**建築師 / 設計協調**。
*   **技術核心**：封裝了原生 API 並加入「批次選取」介面。
*   **可控細節**：圖紙選擇範圍、自定義子資料夾命名。
*   **送審適用度**：適合內部週會、同儕審核，能快速產出整套圖冊。
*   **可擴充性**：高。其 `lib` 庫結構清晰，可進一步整合 DWG 同步輸出。

### 3. [ADSK] 原生匯出 (Export PDF)
*   **適合使用角色**：**建築師 / 專案簽核人**。
*   **技術核心**：官方直接轉換引擎。
*   **可控細節**：**最豐富**。包含字體子集嵌入、顏色管理、點陣化閾值等。
*   **送審適用度**：**官方標準**。在正式送至政府機關或業主簽核時，這是唯一能保證符合 100% 官方標準的路徑。
*   **可擴充性**：**無**。這是一個封閉的 UI 功能，只能由人類手動調整。

### 4. [ADSK] 原生列印 (Virtual Print)
*   **適合使用角色**：**傳統行政人員**。
*   **技術核心**：依賴 Adobe/Bluebeam 等印表機驅動。
*   **適合場景**：當原生 PDF 引擎出現字體渲染錯誤或特殊線寬問題時的「最後防線」。
*   **可擴充性**：極低。

### 5. [pyRevit] 核心印表工具 (Print Sheets)
*   **適合使用角色**：**營造統包 (BIM Coordinator)**。
*   **技術原理**：對 `PrintManager` 的高級包裝。
*   **可控細節**：列印組合 (Print Sets) 管理、圖紙排序。
*   **可擴充性**：中。適合用來處理複雜的「圖冊目錄」列印邏輯。

## 🤖 Agent 執行指南
當收到「出圖」指令時，應優先評估角色與需求：
*   若為**生產/加工**需求 ➡️ 使用 **Potato MCP 途徑**。
*   若為**全套圖冊整理** ➡️ 使用 **guRoo 途徑**。
*   若為**正式發文送審** ➡️ 引導人類使用 **ADSK 原生匯出**。
