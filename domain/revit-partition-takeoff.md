# Revit 輕隔間牆算量作業 SOP

## 1. 目標

將 Revit 專案中屬於輕隔間類型的牆體，依房間彙整成數量計算表。輸出內容應能支援以下用途：

- 依房間列出輕隔間牆類型、牆長、牆高、面積與開口扣除。
- 依 TYPE 分欄彙總輕隔間牆面積。
- 讓房間排序符合建築圖說與送審表格邏輯。
- 與 Revit「輕隔間牆明細表」或既有 Excel 算量表交叉驗證。

## 2. 適用情境

- 專案已建立 Room，且房間編號可對應樓層。
- 牆類型名稱中有穩定的 TYPE 或輕隔間識別規則。
- 需要依房間輸出 CSV 或 Excel 算量表。
- 需要把門窗開口扣除納入輕隔間牆總量。

## 3. 資料需求

### 3.1 房間資料

每個房間至少需要：

- 房間 ElementId
- 房號
- 房名
- 樓層
- 房間中心點或可用於最近房間判斷的定位資訊
- 房間周長或可由邊界推回的周長

### 3.2 牆體資料

每片候選牆至少需要：

- 牆 ElementId
- 牆類型名稱
- 所在樓層
- 牆長
- Revit 牆面積
- 牆高或可由面積反推的有效高度
- 與房間邊界的關聯

輕隔間牆的主要判斷規則是牆類型名稱包含 `TYPE`。若專案使用其他命名方式，例如「輕隔間」、「庫板」、「複壁」，應先在專案內確認命名規則，再加入候選條件。

### 3.3 開口資料

每個牆上開口至少需要：

- 開口族群或類型名稱
- 數量
- 寬度
- 高度
- 所屬牆 ElementId

**開口所屬牆必須來自 Revit host 關係**，例如 `主體 ID` / Host ElementId。不得用「開口座標接近某道 TYPE 牆」推定扣除，因為廁所隔間門、設備開口或鄰牆開口可能靠近輕隔間但實際 host 在非 TYPE 牆上。

若族群或類型名稱包含 `寬 x 高` 尺寸，可解析為開口扣除面積；若無法解析，需保留名稱與數量供人工複核。

## 4. 算量流程

### 4.1 建立房間基準清單

先取得 Revit 房間清單，並依房號推導樓層。若業主或圖說已有主房間清單，應以主清單作為輸出順序，再回填 Revit 實際找到的房間資訊。

### 4.2 取得房間邊界牆

以 Room boundary 為主要來源，收集每個房間周界上的牆段。每個牆段需保留 ElementId，避免同一片牆在同一房間內重複計入。

### 4.3 篩選輕隔間牆

依牆類型名稱篩選 TYPE 牆。建議保留原始牆類型名稱，不要在中途改寫，因為輸出表的 TYPE 分欄需與 Revit 明細表或圖說名稱一致。

### 4.4 計算報表牆高與面積

報表的 `牆高` 欄應優先採用可被審查者理解的幾何高度，而不是逐列用面積反推的係數。一般全高輕隔間牆應依以下口徑：

```text
表內牆高 = 上層樓層標高 - 上層樓板厚度 - 牆基準樓層標高 - 牆基準偏移
```

若牆使用 Revit 的「貼附頂/底」且貼附後幾何高度與一般樓板底高度差異明顯，可視為上方存在斜板、局部降板或特殊頂部條件，報表牆高應採貼附後 Revit 幾何平均高度，並保留高度來源標記。若貼附後高度與一般樓板底高度接近，仍可沿用樓層到上層樓板底高度，讓同樓層同條件的牆高一致。

低矮牆、襯板、腰牆或明顯未到頂牆應保留 Revit 牆本身的 `不連續高度` 或幾何高度，不可強制套用樓層到樓板底高度。預設判斷可用 2.70m 作為全高牆門檻；若未使用「貼附頂/底」且 Revit 牆高比上層樓板底高度低超過約 0.35m，也應視為特殊未到頂牆並保留原牆高。

Revit 原生牆 `面積` 仍可作為總量驗證基準，但不得把 `Revit 面積 / 牆長` 得出的有效係數直接塞入 `牆高` 欄。若需要同時嚴格對齊 Revit 面積，應另增校正欄或檢核表，不應讓牆高欄出現難以解釋的小數。

遇到斜板或局部降板時，不得只靠樓層代表厚度硬猜。「貼附頂/底」可作為第一層判斷訊號；若需要精準拆分局部高度，應升級為 Floor 幾何取樣或 raycast，對牆線取樣並保留命中樓板與樓板底標高證據。

只有在無法取得可信的樓層標高或上層樓板厚度時，才退回使用牆體實際高度。若樓板底高度、Revit 牆高與 Revit 牆面積驗證差異過大，需回頭檢查開口、房間邊界、牆體分段或是否存在斜板/局部降板。

### 4.5 補分配未認領牆

部分複壁、庫板或特殊邊界牆可能不會出現在 Room boundary。這類未被房間認領的 TYPE 牆，可用同樓層最近房間中心點補分配，並在報表或驗證紀錄中標記為補分配項目。

### 4.6 扣除開口

對每片牆彙整門窗與開口族群，依名稱分組後計算扣除面積。若同一開口被多個房間或牆段讀到，需以開口 ElementId 去重。

扣除條件必須是：開口的 host wall ElementId 等於本次計算的 TYPE 牆 ElementId。若開口沒有 host 資料、host 是非 TYPE 牆、或只是在座標上接近 TYPE 牆，均不得扣除；應列為人工複核項或略過。

### 4.7 輸出報表

建議欄位：

| 欄位 | 說明 |
|------|------|
| 編號 | 房號 |
| 空間名稱 | 房間名稱 |
| 空間周長 | 房間周長或主清單周長 |
| 輕隔間牆類型 | TYPE 名稱 |
| 牆長 | 同房間同 TYPE 牆長加總 |
| 牆高 | 全高牆為基準樓層至上層樓板底；低矮/特殊牆為 Revit 牆高 |
| 面積 | 牆面積小計 |
| 開口 | 門窗或開口扣除 |
| 總計 | 扣除開口後的有效面積 |
| TYPE 分欄 | 每個 TYPE 各自分欄，便於加總 |

TYPE 分欄只應列出本次 Revit 專案中實際存在牆實例的輕隔間類型，不應把只有 Wall Type 定義但沒有實體牆的類型帶入報表。表尾需新增 `輕隔間數量總計` 列：左側固定資料欄跨欄置中，右側各 TYPE 欄以 `SUM` 彙總資料列。

## 5. 房間排序規則

排序遵守專案建築樓層語言：

- `FN`、`BS`：基礎或筏基層，排最前。
- `B`：地下層，由深到淺，例如 `B3 -> B2 -> B1`。
- `F`：地上層，由低到高，例如 `F1 -> F2 -> F3`。
- `R`：屋突或屋頂層，由低到高，例如 `R1 -> R2`。

房號可推導樓層，例如：

- `B105 = B1`
- `B201 = B2`
- `F601 = F6`
- `R202 = R2`

## 6. 驗證清單

- 房間數量與主房間清單或 Revit 房間數量一致。
- TYPE 欄位完整展開，且總量接近 Revit 輕隔間牆明細表。
- 樓層排序符合 `FN/BS -> B -> F -> R`。
- 同一牆 ElementId 不在同房間內重複計入。
- 開口扣除均可追溯到 host wall ElementId，且 host wall 是本次計算的 TYPE 牆。
- 抽查至少一個有疑慮房間，確認鄰近但非 host 的門窗沒有被扣除。
- 同一 TYPE 的總面積需與 Revit 牆面積明細表或舊表基準比對；若 `牆長 × 報表牆高` 與 Revit 面積差異明顯，先檢查樓板厚度、特殊未到頂牆、斜板/局部降板與開口扣除，不得直接把面積差異藏進牆高欄。
- 補分配牆有保留可追蹤依據，方便人工複核。

## 7. 常見問題

### 7.1 報表總量與 Revit 明細表不一致

優先檢查 TYPE 篩選條件是否漏掉部分牆類型，其次檢查面積欄位來源是否與 Revit 明細表一致。若明細表使用 Revit 原生牆面積，純粹用 `牆長 × 牆高` 推算可能會出現落差。

若某 TYPE 總量比舊表明顯偏大，先抽查該 TYPE 的 `牆長`、報表牆高、上層樓板厚度、Revit `面積` 與 host-only 開口扣除。舊表若使用 `Revit 面積 / 牆長` 的有效高度，需明確註記為舊口徑；新版報表仍以可解釋的樓板底牆高優先。

### 7.2 房間邊界抓不到複壁或庫板

這通常是 Room boundary 不把該牆視為封閉邊界。可採同樓層最近房間補分配，但需保留標記，避免補分配結果被當成完全自動判定。

### 7.3 房號排序與圖說不一致

先確認房號是否能推導樓層，再確認主房間清單是否有人工排序需求。若圖說排序與建築樓層邏輯不同，應以主清單順序優先。
